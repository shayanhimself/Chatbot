# Chat Engine (`:shared`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `ChatEngine` — the app's sole interface to Claude — in `:shared` commonMain: a stateless, cold-`Flow` streaming client over `POST https://api.anthropic.com/v1/messages` with a hand-rolled SSE parser, typed errors, and a `FakeChatEngine` for downstream feature specs.

**Architecture:** `ChatEngine` is a KMP interface returning `Flow<ChatStreamEvent>`; `ClaudeChatEngine` is the single `internal` implementation over a Ktor `HttpClient`. The client engine is the only platform-specific piece (`expect`/`actual` `createChatHttpClient()`, OkHttp on androidMain). SSE frames are read off `bodyAsChannel()` line-by-line (never buffered), dispatched on the `data` JSON `type` field. Every API and domain failure becomes a `ChatStreamEvent.Failed(ChatError)` — the engine never throws except for structured cancellation. `:shared` exposes exactly two public symbols: the `ChatEngine` interface and `createChatEngine(ApiKeyProvider)`. Spec: `specs/003-chat-engine.md`.

**Tech Stack:** Kotlin 2.4.10, KMP (`androidTarget()` only), Ktor 3.5.1 (`ktor-client-core` common, `ktor-client-okhttp` androidMain, `ktor-client-mock` test), kotlinx.serialization 1.11.0, kotlinx.coroutines 1.11.0 (+ `-test`), kotlin-test. JVM host tests via `:shared:testAndroidHostTest`.

## Deviations from the spec (apply as written; spec text is corrected in Task 8)

1. **No `ContentNegotiation` plugin.** `specs/003-chat-engine.md` lists `ContentNegotiation(json)` inside `installChatDefaults()`. The engine hand-rolls SSE and never calls `body<T>()`, so the only thing ContentNegotiation would do is serialize the request body — which one `Json.encodeToString` call already does, using the same `Json` instance the SSE parser needs anyway. Installing it also appends its own `Accept: application/json` header alongside our explicit `accept: text/event-stream`. Dropping it removes two dependencies (`ktor-client-content-negotiation`, `ktor-serialization-kotlinx-json`) and one ambiguity. `installChatDefaults()` therefore installs `HttpTimeout` only.
2. **`:app` ships the `ChatEngine` Hilt binding only, not the `ApiKeyProvider` binding.** The spec asks for both, but also states no production `ApiKeyProvider` implementation ships in 003 (005 provides a debug stub, 006 the real one). Task 8 adds the `ChatEngine` provider and verifies the app still compiles.
3. **`thinking` is sent as `{ "type": "disabled" }`, not omitted.** See the dedicated section below — omitting it turns adaptive thinking on for the default model. Task 8 corrects the spec.

## Thinking is disabled (deviation 3 — supersedes the spec's "omitted")

The spec says "The `thinking` parameter is omitted." **Omitting is not neutral on the default model.** `ClaudeModel.Default` is `Sonnet` (`claude-sonnet-5`), and on Sonnet 5 a request with no `thinking` field runs **adaptive thinking by default** (Opus 4.8 and Haiku 4.5 run thinking off when omitted — only the default model flips on). With `display` defaulting to `omitted`, that means the engine would pay for thinking tokens on the user's own key, wait through the thinking pause before the first visible token, and then discard every `thinking_delta` — an invisible, billed feature nobody chose.

So this spec sends `thinking: {"type": "disabled"}` explicitly on every request. That makes "no thinking" actually true across all three picker models and keeps the M1 chat turn fast and cheap. Turning thinking on (adaptive, `display: "summarized"`, streamed to the UI) becomes a deliberate product decision in a future spec, not an accidental default. Task 8 corrects the spec line.

```kotlin
package com.shayanaryan.chatbot.shared.chat

import kotlinx.coroutines.flow.Flow

/**
 * The app's sole interface to Claude. Stateless: callers pass the full conversation in, nothing
 * is persisted here.
 *
 * Retained despite the single production implementation so tests can substitute a fake and a
 * future iOS on-device engine can slot in without touching callers.
 */
interface ChatEngine {
    /**
     * Streams one assistant turn.
     *
     * Returns a cold [Flow] — a new HTTP call starts per collection. Main-safe. The flow emits
     * exactly one terminal [ChatStreamEvent.Completed] or [ChatStreamEvent.Failed] and then
     * completes; it never throws for API or domain errors. Only structured cancellation
     * propagates, cancelling the in-flight HTTP call.
     *
     * @param request the full conversation plus model and generation settings.
     * @return the assistant's reply as it arrives.
     */
    fun stream(request: ChatRequest): Flow<ChatStreamEvent>
}
```

package com.shayanaryan.chatbot.shared.chat.sse

import com.shayanaryan.chatbot.shared.chat.SseFixtures
import io.ktor.utils.io.ByteReadChannel
import kotlinx.coroutines.test.runTest
import kotlin.test.Test
import kotlin.test.assertEquals
import kotlin.test.assertTrue

class SseFrameReaderTest {
    private suspend fun framesOf(text: String): List<SseFrame> {
        val collected = mutableListOf<SseFrame>()
        ByteReadChannel(text.encodeToByteArray()).forEachSseFrame { collected += it }
        return collected
    }

    @Test
    fun splits_a_recorded_stream_into_frames() =
        runTest {
            val frames = framesOf(SseFixtures.HAPPY_PATH)

            assertEquals(8, frames.size)
            assertEquals("message_start", frames.first().event)
            assertEquals("message_stop", frames.last().event)
            assertEquals("""{"type":"message_stop"}""", frames.last().data)
        }

    @Test
    fun strips_exactly_one_space_after_the_field_colon() =
        runTest {
            val frames = framesOf("event: ping\ndata: {\"type\": \"ping\"}\n\n")

            assertEquals("ping", frames.single().event)
            assertEquals("""{"type": "ping"}""", frames.single().data)
        }

    @Test
    fun joins_multi_line_data_with_newlines() =
        runTest {
            val frames = framesOf("data: one\ndata: two\n\n")

            assertEquals("one\ntwo", frames.single().data)
        }

    @Test
    fun ignores_comment_lines() =
        runTest {
            val frames = framesOf(": keep-alive\ndata: payload\n\n")

            assertEquals("payload", frames.single().data)
        }

    @Test
    fun emits_a_trailing_frame_that_was_never_blank_terminated() =
        runTest {
            val frames = framesOf("event: message_stop\ndata: {}")

            assertEquals("{}", frames.single().data)

package com.shayanaryan.chatbot.shared.chat.sse

import io.ktor.utils.io.ByteReadChannel
import io.ktor.utils.io.readUTF8Line

/**
 * One server-sent event.
 *
 * @property event the `event:` field, absent when the server omitted it.
 * @property data the `data:` field; multiple data lines are joined with newlines.
 */
internal data class SseFrame(
    val event: String?,
    val data: String,
)

/**
 * Reads the channel line by line, invoking [onFrame] for each complete server-sent event.
 *
 * The channel is consumed as it arrives — never buffered whole — so deltas surface the moment
 * the server writes them. Comment lines and fields other than `event`/`data` are ignored, as are
 * frames that carry no data. Returns when the channel closes.
 */
internal suspend fun ByteReadChannel.forEachSseFrame(onFrame: suspend (SseFrame) -> Unit) {
    var event: String? = null
    val data = StringBuilder()

    suspend fun flush() {
        if (data.isNotEmpty()) {
            onFrame(SseFrame(event, data.toString()))
        }
        event = null
        data.clear()
    }

    while (true) {
        val line = readUTF8Line() ?: break
        when {
            line.isEmpty() -> flush()
            line.startsWith(":") -> Unit
            else -> {
                val separator = line.indexOf(':')
                val field = if (separator == -1) line else line.substring(0, separator)
                val value =
                    if (separator == -1) "" else line.substring(separator + 1).removePrefix(" ")
                when (field) {
                    "event" -> event = value
                    "data" -> {
                        if (data.isNotEmpty()) data.append('\n')
                        data.append(value)
                    }
                }
            }
        }
    }
    flush()
}

private const val ANTHROPIC_VERSION = "2023-06-01"
private const val API_KEY_HEADER = "x-api-key"
private const val SSE_CONTENT_TYPE = "text/event-stream"

/** The one production [ChatEngine]: Ktor over the Anthropic Messages API with hand-rolled SSE. */
internal class ClaudeChatEngine(
    private val client: HttpClient,
    private val keyProvider: ApiKeyProvider,
) : ChatEngine {
    override fun stream(request: ChatRequest): Flow<ChatStreamEvent> =
        flow {
            val apiKey = keyProvider.apiKey()
            val body = chatJson.encodeToString(MessageRequestDto.serializer(), request.toDto())
            client
                .preparePost(MESSAGES_URL) {
                    header(API_KEY_HEADER, apiKey)
                    header(ANTHROPIC_VERSION_HEADER, ANTHROPIC_VERSION)
                    header(HttpHeaders.Accept, SSE_CONTENT_TYPE)
                    contentType(ContentType.Application.Json)
                    setBody(body)
                }.execute { response ->
                    emitStream(response.bodyAsChannel())
                }
        }
}

/**
 * Folds SSE frames into stream events. Accumulates usage and stop reason as they arrive, emits a
 * [ChatStreamEvent.Delta] per `text_delta`, and guarantees exactly one terminal event: a stream
 * that ends without `message_stop` is a truncation, which is [ChatError.Unexpected].
 */
private suspend fun FlowCollector<ChatStreamEvent>.emitStream(channel: ByteReadChannel) {
    var inputTokens = 0
    var outputTokens = 0
    var stopReason = StopReason.Unknown
    var terminated = false

    channel.forEachSseFrame { frame ->
        if (terminated) return@forEachSseFrame
        val payload = frame.data.toJsonObjectOrNull()
        if (payload == null) {
            emit(ChatStreamEvent.Failed(ChatError.Unexpected))
            terminated = true
            return@forEachSseFrame
        }
        when (payload.string("type")) {
            "message_start" ->
                inputTokens = payload.obj("message")?.obj("usage")?.int("input_tokens") ?: 0

            "content_block_delta" -> {
                val delta = payload.obj("delta")
                if (delta?.string("type") == "text_delta") {
                    delta.string("text")?.let { emit(ChatStreamEvent.Delta(it)) }
                }
            }

            "message_delta" -> {
                payload.obj("delta")?.string("stop_reason")?.let { stopReason = it.toStopReason() }
                payload.obj("usage")?.int("output_tokens")?.let { outputTokens = it }
            }

            "message_stop" -> {
                emit(
                    ChatStreamEvent.Completed(stopReason, TokenUsage(inputTokens, outputTokens)),
                )
                terminated = true
            }

            "error" -> {
                emit(ChatStreamEvent.Failed(payload.obj("error")?.string("type").toChatError()))
                terminated = true
            }
        }
    }

    if (!terminated) {
        emit(ChatStreamEvent.Failed(ChatError.Unexpected))
    }
}

private fun String.toJsonObjectOrNull(): JsonObject? =
    try {
        chatJson.parseToJsonElement(this) as? JsonObject
    } catch (_: IllegalArgumentException) {
        // SerializationException extends IllegalArgumentException, so this covers both.
        null
    }

private fun JsonObject.obj(name: String): JsonObject? = this[name] as? JsonObject

private fun JsonObject.string(name: String): String? = (this[name] as? JsonPrimitive)?.contentOrNull

private fun JsonObject.int(name: String): Int? = (this[name] as? JsonPrimitive)?.intOrNull

private fun String.toStopReason(): StopReason =
    when (this) {
        "end_turn" -> StopReason.EndTurn
        "max_tokens" -> StopReason.MaxTokens
        "stop_sequence" -> StopReason.StopSequence
        "refusal" -> StopReason.Refusal
        else -> StopReason.Unknown
    }

/** Maps an Anthropic `error.type` string onto a [ChatError]. */
private fun String?.toChatError(): ChatError =
    when (this) {
        "authentication_error", "permission_error" -> ChatError.Authentication
        "rate_limit_error" -> ChatError.RateLimited(null)
        "overloaded_error" -> ChatError.Overloaded
        "invalid_request_error", "not_found_error", "request_too_large" -> ChatError.InvalidRequest
        "api_error" -> ChatError.Server
        else -> ChatError.Unexpected
    }

private const val STATUS_OVERLOADED = 529

/** Maps a non-2xx response onto a [ChatError], reading the `retry-after` hint when present. */
private fun HttpResponse.toChatError(): ChatError =
    when (status.value) {
        401, 403 -> ChatError.Authentication
        429 -> ChatError.RateLimited(headers[RETRY_AFTER_HEADER]?.toIntOrNull())
        STATUS_OVERLOADED -> ChatError.Overloaded
        in 400..499 -> ChatError.InvalidRequest
        in 500..599 -> ChatError.Server
        else -> ChatError.Unexpected
    }

/** Maps a transport or decoding failure onto a [ChatError]. Order matters: timeouts are IOExceptions. */
private fun Throwable.toChatError(): ChatError =
    when (this) {
        is HttpRequestTimeoutException, is SocketTimeoutException, is ConnectTimeoutException ->
            ChatError.Timeout

        is IOException -> ChatError.Network
        else -> ChatError.Unexpected
    }
